## Cross-Feature Awareness Rules

These rules apply throughout the entire development process:

1. **Before starting a Feature**: Read `roadmap.md` to understand:
   - Which Features this one depends on (prerequisites)
   - Which Features depend on this one (downstream impact)
   - Current Feature's Tier and Release Group

2. **When modifying entities or APIs**: Check the registries for cross-Feature usage:
   - `entity-registry.md` → "Used by Features" column
   - `api-registry.md` → "Cross-Feature Consumers" column
   - If other Features use it, ensure backward compatibility

3. **After completing a Feature**: Review downstream Features' pre-contexts:
   - Read their "For /speckit.plan" → dependency sections
   - Verify your finalized contracts satisfy their expectations
