# Concern: i18n

> Internationalization and localization.
> Applies when the project uses i18n frameworks (i18next, react-intl, vue-i18n, etc.).
> Module type: concern

---

## S0. Signal Keywords

> See [`shared/domains/concerns/i18n.md`](../../../shared/domains/concerns/i18n.md) § Signal Keywords

---

## S1. SC Generation Rules

### Required SC Patterns
- All user-visible text: must come from locale files, not hardcoded strings
- Locale switching: specify trigger + expected language change + persistent preference

### SC Anti-Patterns (reject)
- "Text is translated" — must specify which locale files and key naming convention

## S7. Bug Prevention Rules

When this concern is active, enforce:
- i18n Completeness Check (per-task): after each implementation task, verify new t() keys exist in ALL locale files. See `injection/implement.md` § Step 1b
- i18n Coverage Lint (verify): cross-check code keys vs locale keys vs locale-to-locale consistency. See `verify-phases.md` § Phase 1 Step 4
